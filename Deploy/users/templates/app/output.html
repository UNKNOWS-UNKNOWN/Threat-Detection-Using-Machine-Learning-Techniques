🔗 ARP Poisoning

Aspect Details
❓Description ARP Poisoning involves an attacker sending falsified ARP messages to a local network, linking their MAC address to an IP address of a legitimate device.
⚠️Symptoms Network traffic may be intercepted, data can be stolen, and connections might be rerouted.
🚨Causes Vulnerable network configurations, lack of encryption, and insufficient monitoring.
🛡️Prevention Implement packet filtering, use static ARP entries, and regularly monitor network traffic.
🚧Precaution Educate users on secure network practices and deploy ARP spoofing detection tools.
{% elif result == 'DDOS_Slowloris' %}

💥 DDoS Slowloris

Aspect Details
❓Description The Slowloris attack is a type of Denial of Service (DoS) attack that sends incomplete HTTP requests to exhaust server resources.
⚠️Symptoms Server slows down or crashes, legitimate users experience delays or service interruptions.
🚨Causes Exploits vulnerabilities in the HTTP protocol, lack of proper server configuration.
🛡️Prevention Implement timeouts on HTTP connections, use load balancers, and monitor server traffic.
🚧Precaution Deploy a Web Application Firewall (WAF) and keep server software updated.
{% elif result == 'DOS_SYN_Hping' %}

⚡ DoS SYN Hping

Aspect Details
❓Description SYN Flood is a type of DoS attack that overwhelms a server with SYN packets, causing it to become unresponsive.
⚠️Symptoms Server becomes unresponsive, legitimate users are unable to access services.
🚨Causes Exploitation of the TCP handshake process, lack of SYN flood protection.
🛡️Prevention Configure SYN cookies, limit connection rates, and use intrusion prevention systems (IPS).
🚧Precaution Monitor network traffic for unusual patterns and employ redundancy in network architecture.
{% elif result == 'Metasploit_Brute_Force_SSH' %}

🔓 Metasploit Brute Force SSH

Aspect Details
❓Description Brute force attack using Metasploit to gain unauthorized access to SSH by systematically trying various password combinations.
⚠️Symptoms Multiple failed login attempts, potential unauthorized access to systems.
🚨Causes Weak or default passwords, lack of account lockout policies.
🛡️Prevention Use strong, unique passwords, implement two-factor authentication, and limit SSH access.
🚧Precaution Monitor for unusual login activity and regularly update security protocols.
{% elif result == 'MQTT_Publish' %}

📡 MQTT Publish

Aspect Details
❓Description MQTT Publish attack involves sending unauthorized messages to an MQTT broker, potentially leading to data manipulation or network disruption.
⚠️Symptoms Unexpected messages in the network, disrupted IoT device functionality.
🚨Causes Weak authentication, unsecured MQTT broker configurations.
🛡️Prevention Implement strong authentication, encrypt MQTT traffic, and secure broker access.
🚧Precaution Regularly audit MQTT configurations and monitor for unauthorized access.
{% elif result == 'NMAP_FIN_SCAN' %}

🔍 NMAP FIN Scan

Aspect Details
❓Description NMAP FIN Scan sends packets with a FIN flag to check open ports without establishing a full TCP connection.
⚠️Symptoms Stealthy scanning activity, potential reconnaissance by attackers.
🚨Causes Exploitation of TCP/IP stack vulnerabilities, lack of network monitoring.
🛡️Prevention Use intrusion detection systems (IDS), block suspicious IP addresses, and monitor network activity.
🚧Precaution Regularly audit network security configurations and educate staff on security awareness.

🖥️ NMAP OS Detection

Aspect Details

⚠️Symptoms Increased network scanning activity, potential profiling of devices.

🛡️Prevention Implement strict firewall rules, use network segmentation, and hide system details.

{% elif result == 'NMAP_TCP_scan' %}

🔍 NMAP TCP Scan

Aspect Details
❓Description NMAP TCP Scan is a common method for scanning open ports on a network to identify potential vulnerabilities.
⚠️Symptoms Unusual network traffic, potential identification of open ports.
🚨Causes Vulnerable network services, lack of port security.
🛡️Prevention Limit open ports, use firewalls to block unauthorized scans, and monitor network traffic.

🔍 NMAP UDP Scan

Aspect Details
❓Description NMAP UDP Scan checks for open UDP ports, which are often overlooked but can be vulnerable to attacks.
⚠️Symptoms Unusual UDP traffic, identification of vulnerable services.
